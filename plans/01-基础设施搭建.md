## 1.1 概述

本阶段是整个 Flutter 重写项目的**地基工程**。参考 `legado-with-MD3` 项目的分层架构（Clean Architecture），在 Flutter 中搭建完整的项目骨架。

### 技术栈决策（已确认）

| 决策项 | 选择 | 说明 |
|-------|------|------|
| Flutter 版本 | **3.41.5** (Dart 3.11.3) | 当前最新稳定版，Impeller 已全面接管 Android 渲染 |
| 目标平台 | **Android + iOS** | 双端支持，Android Monet 取色，iOS 降级到 `ColorScheme.fromSeed()` |
| 架构模式 | **Clean Architecture 三层** | data / domain / ui 分离 |
| 状态管理 | **flutter_riverpod 3.3.2** | 替代 Jetpack Compose ViewModel，手动写 Provider（不依赖 codegen） |
| 依赖注入 | **get_it 8.3.0** | 替代 Koin，手动注册模块 |
| 数据库 | **drift 2.34.1** | 替代 Room + KSP，全新数据库（不兼容旧 94 次迁移） |
| 网络请求 | **dio 5.10.0** | 替代 OkHttp 5.x |
| JS 引擎 | **flutter_js 0.8.7** (QuickJS) | 替代 Rhino，Android 用 QuickJS，iOS 用 JavaScriptCore |
| 后台 WebView | **flutter_inappwebview 6.1.5** (HeadlessInAppWebView) | 替代 BackstageWebView，双平台均支持 |
| 路由导航 | **go_router 17.3.0** | 替代 Navigation Compose |
| 数据模型 | **freezed + json_serializable** | 替代 Kotlin Serialization |
| 数据库策略 | **全新数据库** | 不兼容旧版 Legado 数据库文件，用户需重新导入书源 |

### 步骤 1.7：JS 引擎 — QuickJS 集成 ✅（已完成）

**做什么**：
- 集成 `flutter_js 0.8.7`（Android 用 QuickJS，iOS 用 JavaScriptCore）
- 封装 `JsEngine` 驱动类
- 实现脚本编译缓存
- 实现 JS 扩展函数注册框架

**实际完成情况**：
- js_engine_wrapper.dart — QuickJS 封装 + bridge.js 桥接层（128个 java.* 函数注入）
- js_extensions.dart — 所有 onMessage 处理器（网络/文件/编解码/加解密/时间/URL/文本/工具）
- compiled_script.dart — LRU 脚本编译缓存（最多 16 项）
- shared_js_scope.dart — 书源 JS 库共享作用域
- headless_webview.dart — HeadlessInAppWebView 后台渲染（普通/嗅探/拦截三种模式）
- `dart analyze` 零问题

**关于 Rhino vs QuickJS 的关键结论**（经原项目代码分析确认）：
- 原项目中 JS 脚本**通过 `java.xxx()` 固定接口与原生端交互**，而非直接访问 Java 类
- 不存在 `Java.type()` 或 `Packages` 等传统 Rhino Java 互操作的调用
- 这意味着我们只需要在 QuickJS 中注册同样的一组 `java.*` 函数，即可实现相同的桥接模式
- **核心工作**：将 `JsExtensions.kt` 中的 100+ 个函数逐个注册到 QuickJS 上下文

**✅ 已完成**：
- [x] js_engine_wrapper.dart — QuickJS 引擎初始化和执行封装 + bridge.js
- [x] js_extensions.dart — `java.*` 扩展函数注册（128个函数全部覆盖，含网络/编解码/加解密/时间/URL/文本/工具）
- [x] compiled_script.dart — 脚本编译缓存（LRU，最多 16 项）
- [x] shared_js_scope.dart — 书源 JS 库共享作用域机制
- [x] headless_webview.dart — HeadlessInAppWebView 后台 WebView 封装（含嗅探/拦截模式）

**关键依赖风险**：

| 原项目依赖 | Flutter 替代方案 | 风险等级 | 说明 |
|-----------|----------------|---------|------|
| Mozilla Rhino 1.8.1 | flutter_js 0.8.7 (QuickJS/JavaScriptCore) | 🟡 **中（已降级）** | ES2020 支持完整，缺失 Rhino 特有 API 但原项目未使用 Java 反射互操作，主要工作是逐个移植 JsExtensions 函数 |
| Rhino ClassShutter | flutter_js 内置沙箱 | 🟢 低 | QuickJS 本身有安全沙箱机制 |
| Rhino 编译缓存 | Dart 内存缓存（Map + LRU） | 🟢 低 | 简单实现即可，最多缓存 16 个脚本 |
| JS 扩展函数注入 | QuickJS `onMessage` API | 🟡 中 | 通过 `javascriptRuntime.onMessage()` 注册，与 Rhino 的 bindings 注入方式不同，需适配 |
