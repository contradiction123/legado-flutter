# 第 2 步：书源与规则系统（Flutter 重写 Phase 2）

> **预计工期**：3 周  
> **对应原计划文档**：Phase 2（书源与规则系统）  
> **核心目标**：移植完整的书源规则引擎、JS 扩展、WebBook 引擎、书源管理 UI，这是整个应用的灵魂

---

## 2.1 概述

本阶段是整个 Flutter 重写项目最**核心且最具技术挑战**的环节。书源系统是 Legado 的生命线，规则引擎决定了书源兼容性。

原项目的书源系统位于以下位置：
- 规则引擎：`legado-with-MD3/app/src/main/java/io/legado/app/model/analyzeRule/`
- 网络书籍引擎：`legado-with-MD3/app/src/main/java/io/legado/app/model/webBook/`
- 书源管理 UI：`legado-with-MD3/app/src/main/java/io/legado/app/ui/book/source/`
- JS 引擎模块：`legado-with-MD3/modules/rhino/` 和 `legado-with-MD3/app/src/main/java/io/legado/app/help/rhino/`

## 2.2 详细步骤

### 步骤 2.1：JS 扩展函数移植（最关键子步骤）

**做什么**：
- 分析原版 Rhino 注入到 JS 环境的所有扩展函数
- 在 QuickJS 上下文中注册同名扩展
- 实现 `java` 对象的 polyfill
- 实现网络请求、编解码、加密等工具函数

**参考的原项目代码**：
- `legado-with-MD3/modules/rhino/src/main/java/com/script/rhino/RhinoExtensions.kt` — Rhino 引擎扩展定义
- `legado-with-MD3/app/src/main/java/io/legado/app/help/rhino/NativeBaseSource.kt` — Native 对象桥接（拦截 Java 对象 set* 属性）
- `legado-with-MD3/modules/rhino/src/main/java/com/script/rhino/RhinoTopLevel.kt` — 顶层作用域（JS 全局对象定义）
- `legado-with-MD3/modules/rhino/src/main/java/com/script/rhino/RhinoClassShutter.kt` — 类安全快门（哪些 Java 类允许 JS 访问）

**Flutter 重写内容**：
- 完善 `lib/engine/js_engine/js_extensions.dart` — 注册以下扩展 API：
  - `java.get(url)` / `java.post(url, body)` — HTTP 请求（调用 dio）
  - `java.base64Encode(str)` / `java.base64Decode(str)` — Base64 编解码
  - `java.md5Encode(str)` — MD5 加密
  - `java.aesEncrypt(key, data)` / `java.aesDecrypt(key, data)` — AES 加密解密
  - `java.mergeJson(json1, json2)` — JSON 合并
  - `java.ajReport(key, value)` — AJ 报告（可选）
  - `java.formatDate(date, pattern)` — 日期格式化
  - `java.regexMatch(str, pattern)` — 正则匹配
  - 以及原版 Rhino 中 `org.jsoup`、`com.google.gson`、`java.net.URLEncoder` 等常用 Java 类的 Dart 替代

**当前步骤实现的功能**：
- ✅ `java.get()` / `java.post()` 网络请求
- ✅ Base64、MD5、AES 编解码
- ✅ JSON 操作工具
- ✅ 日期格式化
- ✅ URL 编码/解码
- ✅ 正则匹配封装

**关键依赖风险**：

| 原项目 API | Flutter 替代 | 风险等级 | 说明 |
|-----------|-------------|---------|------|
| `java.get()` 系列 | dio 封装 | 🟡 中 | 需要确保异步 JS 调用正确 |
| `java.base64*` | dart:convert | 🟢 低 | 原生支持 |
| `org.jsoup.*` | 无 JS 等价 | 🟡 中 | 书源 JS 中若直接使用 Java jsoup 类，需要 polyfill 或 Dart 端支持 |
| `Java.type()` | ❌ **无替代** | 🔴 **高** | Rhino 特有的 Java 类型系统，QuickJS 不支持。需要为每个常用 Java 类型提供 Dart polyfill |
| `Packages.*` | ❌ **无替代** | 🔴 **高** | 另一个 Rhino 特有 API |

### 步骤 2.8：WebBook 网络书籍引擎

**做什么**：
- 移植网络书籍引擎，实现完整的书源规则执行流程
- 实现搜索、书籍信息获取、目录获取、正文获取、发现搜索五大核心功能

**参考的原项目代码**：
- `legado-with-MD3/app/src/main/java/io/legado/app/model/webBook/WebBook.kt` — 网络书籍引擎主类（最核心的文件）
- `legado-with-MD3/app/src/main/java/io/legado/app/model/webBook/BookInfo.kt` — 书籍信息数据类
- `legado-with-MD3/app/src/main/java/io/legado/app/model/webBook/BookList.kt` — 搜索结果列表数据类
- `legado-with-MD3/app/src/main/java/io/legado/app/model/webBook/BookChapterList.kt` — 章节列表数据类
- `legado-with-MD3/app/src/main/java/io/legado/app/model/webBook/BookContent.kt` — 正文内容数据类

**Flutter 重写内容**：
- 创建 `lib/engine/web_book/web_book.dart` — 核心引擎，实现以下方法：

  **`searchBooks(BookSource, keyword, page)`**：
  1. 从书源获取 `searchUrl` 模板
  2. 通过 AnalyzeUrl 拼接搜索 URL
  3. 发送 HTTP 请求（dio）
  4. 通过 AnalyzeRule 执行 `ruleSearch` 解析结果
  5. 返回 `List<SearchBook>`

  **`getBookInfo(BookSource, book)`**：
  1. 请求书籍详情页 URL
  2. 执行 `ruleBookInfo` 提取书名、作者、封面、简介
  3. 返回填充后的 `Book` 对象

  **`getChapterList(BookSource, book)`**：
  1. 请求目录页 URL
  2. 执行 `ruleToc` 提取章节列表
  3. 返回 `List<BookChapter>`

  **`getBookContent(BookSource, chapter)`**：
  1. 请求正文页 URL
  2. 执行 `ruleContent` 提取正文文本
  3. 返回 `String` 内容

  **`explore(BookSource, url)`**：
  1. 使用书源的 `exploreUrl` 模板
  2. 解析发现页规则
  3. 返回发现列表

**当前步骤实现的功能**：
- ✅ 多书源并发搜索
- ✅ 书籍信息自动获取
- ✅ 章节列表获取
- ✅ 正文内容获取
- ✅ 发现页面浏览

**关键依赖风险**：

| 原项目依赖 | Flutter 替代方案 | 风险等级 | 说明 |
|-----------|----------------|---------|------|
| WebBook 核心逻辑 | Dart 重写 | 🟡 中 | 逻辑复杂，涉及多层规则嵌套调用的边界情况处理 |

## 2.5 下一阶段展望

完成本阶段后，项目拥有了最核心的**书源系统**：
- 能够在多个书源中搜索书籍
